# Платёжный Компас

Платёжный Компас — Telegram-бот для финансовой дисциплины. Он помогает не допускать просрочек по кредитам, рассрочкам и обязательным платежам: пользователь добавляет платежи и доходы, а бот после каждого полученного дохода показывает, сколько нужно отложить и сколько можно безопасно тратить.

## Статус MVP

На текущем этапе активен только базовый функционал: доходы, платежи, резервирование, ближайшие платежи, прогресс, редактирование и возврат после периода бездействия. Расширенные функции временно скрыты до стабилизации финансового ядра.

## Возможности MVP

- добавление кредитов, рассрочек, аренды, ЖКХ и других платежей;
- добавление ожидаемых и полученных доходов;
- просмотр списка добавленных доходов;
- просмотр итоговой суммы доходов;
- просмотр итоговой суммы ближайших платежей;
- расчёт суммы, которую нужно отложить с дохода;
- расчёт суммы, которую можно тратить;
- список ближайших платежей;
- прогресс по долгам;
- редактирование доходов и платежей;
- изменение статуса дохода с пересчётом рекомендаций;
- возврат после периода бездействия;
- массовое обновление старых платежей и доходов;
- правило: сегодняшние события не изменяются автоматически;
- ежедневные уведомления через APScheduler;
- хранение данных в SQLite;
- безопасная отмена текущего действия;
- автоматический переход в другой раздел из любого шага ввода.

В текущей MVP-версии платёж считается закрытым для планирования, когда бот зарезервировал на него полную сумму. Отдельная отметка оплаты временно скрыта, чтобы не усложнять сценарий.

## Функции, временно замороженные до стабилизации ядра

- финансовый статус;
- копилка/накопления;
- минимум на жизнь;
- план до зарплаты;
- что если купить;
- ручная отметка оплаты платежа.

Код, таблицы и данные этих функций сохраняются, но пользовательский доступ через меню и команды временно отключён.

## Режим разработки для функции «Я вернулся»

Для удобного тестирования можно включить постоянное отображение кнопки:

```env
IM_BACK_ALWAYS_VISIBLE=true
```

В этом режиме кнопка “🔄 Я вернулся” всегда отображается в самом верху меню.

Для продуктового режима:

```env
IM_BACK_ALWAYS_VISIBLE=false
```

Тогда кнопка будет появляться только после 14+ дней отсутствия пользователя.

## DEV-команды для тестирования

В режиме разработки можно включить:

```env
DEV_MODE=true
```

После этого доступны скрытые команды:

`/dev_reset_state` — сбрасывает статусы доходов и очищает резервы/оплаты, но оставляет доходы и платежи.

`/dev_clear_all` — полностью удаляет доходы, платежи, резервы и оплаты текущего пользователя.

Эти команды не отображаются в `/help` и не предназначены для продакшена.

## Архитектура

Все файлы проекта должны быть сохранены в UTF-8.


Бизнес-логика не хранится в Telegram handlers. Обработчики только принимают ввод, вызывают services и отправляют готовые сообщения. Расчёты лежат в `app/calculations.py`, работа с БД в `app/repositories`, продуктовые сценарии в `app/services`, форматирование текста в `app/formatters.py`.

Позже можно добавить Telegram Mini App и REST API по той же схеме:

`Mini App -> REST API -> Services -> Repositories -> Database`

## Как создать бота

1. Открой Telegram и найди `@BotFather`.
2. Выполни команду `/newbot`.
3. Задай имя и username бота.
4. Скопируй токен и вставь его в `.env`.

## Локальный запуск

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Установка и запуск:

```bash
pip install -r requirements.txt
cp .env.example .env
```

Вставь `BOT_TOKEN` в `.env`, затем:

```bash
python -m app.main
```

Таблицы создаются автоматически при запуске. Alembic в MVP не используется.

Если вы уже использовали базу с прежним именем и хотите сохранить данные, переименуйте файл базы в payment_compass.db или обновите DATABASE_URL.

## Docker

```bash
cp .env.example .env
docker compose up --build
```

## Команды

- `/start` — запустить бота;
- `/menu` — главное меню;
- `/add_income` — добавить доход;
- `/incomes` — список доходов;
- `/add_obligation` — добавить платёж;
- `/spend` — сколько можно тратить;
- `/payments` — ближайшие платежи;
- `/progress` — прогресс долгов;
- `/im_back` — я вернулся;
- /help — помощь;
- /cancel — отменить текущее действие.

## Ограничения MVP

- валюта только RUB;
- нет банковских интеграций;
- нет внешних платёжных API;
- нет Mini App и веб-интерфейса;
- авторизация только по `telegram_id`;
- рекомендации являются расчётом для личного планирования.
- накопления считаются внутри бота и не связаны с банковским счётом;
- бот не переводит деньги автоматически;
- “Копилка” показывает рекомендованную сумму для откладывания.
- минимум на жизнь является ориентиром для личного планирования;
- бот не контролирует реальные траты пользователя;
- план до зарплаты рассчитывается на основе данных, которые пользователь сам внёс.
- функция “Что если купить?” является симуляцией;
- она не списывает деньги и не меняет реальные данные;
- точность зависит от доходов и платежей, которые пользователь внёс в бота.

## Дисклеймер

Бот помогает с личным планированием и не является финансовой, юридической или инвестиционной консультацией.

## Дополнительно: редактирование «Уже отложено»

Возможности MVP дополнены:

- ручное редактирование суммы «Уже отложено» по каждому платежу;
- автоматический пересчёт финансовых данных после изменения суммы «Уже отложено».

В разделе «Редактировать платежи» можно выбрать конкретный платёж и изменить сумму «Уже отложено». Бот не перезаписывает платёж и не удаляет историю, а создаёт корректирующую запись в истории отложенных сумм.

## Проверка FSM-навигации

1. Нажать «➕ Добавить доход», на вопрос «Как назвать доход?» нажать «🏠 В меню»: бот показывает главное меню, доход не создаётся, текст кнопки не становится названием дохода.
2. Нажать «➕ Добавить доход», на вопрос «Как назвать доход?» нажать «❌ Отменить действие»: бот пишет «Действие отменено.» и показывает меню.
3. Нажать «➕ Добавить платёж», на вопрос «Как называется платёж?» нажать «🏠 В меню»: бот показывает меню, платёж не создаётся, текст кнопки не становится названием платежа.
4. Нажать «➕ Добавить платёж», на вопрос «Как называется платёж?» нажать «❌ Отменить действие»: бот отменяет действие, платёж не создаётся.
5. Нажать «➕ Добавить доход», затем «⚙️ Настройки»: настройки открываются, сценарий добавления дохода прерывается.
6. Нажать «➕ Добавить платёж», затем «💰 Сколько можно тратить?»: расчёт открывается, сценарий платежа прерывается.

## Ручной smoke-check упрощённого MVP

1. `/start` показывает упрощённое меню.
2. В меню нет:
   - 📍 Финансовый статус
   - 🛒 Что если купить?
   - 📆 План до зарплаты
   - 🛟 Минимум на жизнь
   - 🏦 Накопления
3. «➕ Добавить доход» работает.
4. «➕ Добавить платёж» работает.
5. «📅 Ближайшие платежи» работает.
6. «💰 Сколько можно тратить?» работает.
7. В меню нет кнопки «✅ Отметить оплату».
8. Полностью зарезервированный платёж исчезает из ближайших незакрытых платежей.
9. Следующий регулярный экземпляр платежа появляется после закрытия текущего резервами.
10. «✏️ Редактировать» работает.
11. «❌ Отменить действие» работает.
12. «🏠 В меню» работает.
13. Если вручную отправить «📍 Финансовый статус» или `/status`, бот отвечает, что функция временно отключена.
14. Если вручную отправить «✅ Отметить оплату», бот отвечает, что функция временно отключена.
15. Если вручную отправить `/im_back`, бот сначала показывает предварительный расчёт и не меняет данные без подтверждения.
